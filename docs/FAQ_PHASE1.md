# Phase 1 FAQ — Markov, 보상 설계

RL 학습 중 자주 나오는 질문과 답변.

---

## Q1. Markov — "과거 history를 아는 게 AI에서 중요한 거 아닌가? memory, 기억 말이야. 왜 RL은 달라?"

### Markov가 말하는 것

**"다음 상태는 현재 상태와 행동에만 의존한다"** = 전이(transition)가 `P(s'|s,a)`로만 결정된다는 뜻.

과거 경로 `s₀,a₀,s₁,a₁,...`는 **전이 확률**에 영향을 안 준다. 그래서 "과거 경로를 저장할 필요가 없다"는 말은 **"과거를 무시하라"**가 아니라, **"현재 상태 s만 있으면 전이와 최적 결정에 필요한 정보가 다 들어있다"**는 뜻.

### 상태에 메모리를 넣을 수 있다

핵심: **"상태"를 어떻게 정의하느냐.**

- 상태 = (위치, 속도) → Markov
- 상태 = (위치, 속도, "마지막 보상 이후 스텝 수") → 이것도 Markov

**과거 정보를 "상태"에 인코딩**하면 Markov를 유지한다. "과거를 저장할 필요가 없다" = **상태 표현에 이미 필요한 과거가 들어가 있다**는 뜻.

### 메모리가 필요한 경우: POMDP

메모리가 꼭 필요한 건 **부분 관측(POMDP)**일 때.

- **관측(observation)**이 **전체 상태(state)**를 다 보여주지 않을 때
- 예: 상대 카드를 못 보는 카드 게임, 일부만 보이는 맵

이때는 현재 관측만으로 "진짜 상태"를 알 수 없어서, **과거 관측/행동 이력을 모아서** 상태를 추정해야 함. LSTM, Transformer, belief state가 그 역할.

**정리**: 완전 관측 MDP면 현재 상태에 다 들어있음. 부분 관측이면 과거(메모리)가 필요.

## Q2. 누적 보상 — "시간에 비례하는 거 아닌가? 항상 기울어진 운동장에서 싸우는 거랑 마찬가지 아닌가?"

### 맞는 부분

에피소드가 길수록 보상 합이 더 쌓인다. 스텝당 -0.01이면 100스텝 = -1.0, 10스텝 = -0.1. **"시간에 비례"**라는 말은 그 의미에서 맞다.

### 그런데 이건 "불공정"이 아니라 "설계"다

우리가 원하는 건 **목표를 빨리 도달**하는 것. 그래서 스텝이 많을수록 누적 보상이 나빠지도록 설계한 것.

- 8스텝에 도달: +1 - 0.08 = 0.92
- 50스텝에 도달: +1 - 0.5 = 0.5

같은 규칙으로 평가하되, **효율적인 경로**가 더 높은 보상을 받도록 만든 것.

### 할인(discount) γ의 역할

실제로는 **할인 누적 보상**을 쓴다:

```
G_t = r_t + γ·r_{t+1} + γ²·r_{t+2} + ...
```

γ < 1이면 **가까운 미래**가 더 중요해짐. 먼 미래 보상은 0에 가깝게 줄어듦. 그래서 "시간이 길면 무조건 유리"가 아니라 **언제** 보상을 받는지가 중요해짐.

### "기울어진 운동장" 비유

- **같은 규칙**으로 모든 에이전트를 평가하므로 "공정"하다.
- 다만 **보상 설계가 학습 방향을 결정**하므로, "어떤 행동이 유리한 운동장"인지는 우리가 보상으로 설계한다.
- 스텝 패널티를 넣으면 → "빨리 끝내는 쪽"이 유리한 운동장
- 안 넣으면 → "천천히 가도 상관없는" 운동장

즉, **의도적으로 기울어진 보상 구조**를 만드는 것.

## 요약

| 질문 | 핵심 |
|------|------|
| **Markov vs 메모리** | Markov = 전이에 필요한 정보가 현재 상태에 다 있다. 상태에 과거를 인코딩하면 됨. 부분 관측(POMDP)일 때만 별도 메모리(히스토리)가 필요. |
| **누적 보상과 시간** | 에피소드가 길수록 누적 보상이 변하는 건 맞지만, 그건 "빨리 끝내라"는 설계. 할인 γ로 가까운 미래를 더 중요하게 두고, 보상 구조 자체가 학습 방향을 결정함. |
